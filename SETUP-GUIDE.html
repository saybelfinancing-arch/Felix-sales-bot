Felix Slack Bot

Setup Guide · SBL IT Platforms Co., Ltd. · Sales Agent
Step 01

Create a Slack App

Go to https://api.slack.com/apps and click Create New App → From Scratch.

Name it Felix and select your SBL IT Platforms workspace.

Then configure the following permissions under OAuth & Permissions → Bot Token Scopes:

ScopeWhy needed
chat:writePost messages as Felix
chat:write.publicPost to channels without being invited
channels:historyRead channel messages
im:historyRead direct messages
im:writeSend DMs to users
app_mentions:readDetect when @Felix is mentioned
commandsEnable /felix slash command

Click Install to Workspace and copy your Bot User OAuth Token (starts with xoxb-).

Also copy the Signing Secret from Basic Information.

Step 02

Deploy Felix to Railway (free)

Railway is the easiest way to host Felix's backend. Go to https://railway.app and sign up free.

Option A — Deploy from GitHub (recommended):

# 1. Create a GitHub repo and push the felix-slack-bot folder git init git add . git commit -m "Felix Sales Bot v1.0" git remote add origin https://github.com/YOUR_USERNAME/felix-slack-bot.git git push -u origin main

Then in Railway: New Project → Deploy from GitHub repo → select your repo.

Option B — Deploy from local folder:

# Install Railway CLI npm install -g @railway/cli # Login and deploy from the felix-slack-bot folder cd felix-slack-bot railway login railway init railway up

After deploy, Railway gives you a public URL like: https://felix-slack-bot.up.railway.app

Step 03

Set Environment Variables

In Railway → your project → Variables, add these three:

ANTHROPIC_API_KEY = sk-ant-your-key-from-console.anthropic.com SLACK_BOT_TOKEN = xoxb-your-bot-token-from-slack SLACK_SIGNING_SECRET = your-signing-secret-from-slack
⚠️ Security: Never share these values publicly or commit them to Git. The .env.example file is safe to commit — your actual .env file is not.
Step 04

Connect Slack to Your Server

Back in your Slack App settings:

1. Event Subscriptions → Enable Events → Request URL:

https://felix-slack-bot.up.railway.app/slack/events

Slack will send a challenge — your server responds automatically. You'll see a green ✓.

Subscribe to these Bot Events:

message.channels message.im app_mention

2. Slash Commands → Create New Command:

FieldValue
Command/felix
Request URLhttps://felix-slack-bot.up.railway.app/slack/commands
DescriptionTalk to Felix, your SBL Sales Agent
Usage hintgenerate leads | write proposal | follow up

3. Reinstall your app to apply all changes — click Reinstall to Workspace.

Step 05

Add Felix to Your Sales Channel

In Slack, create a channel called #felix-sales (or any name you like).

Then type in the channel:

/invite @Felix

Felix is now a member of the channel and will respond to any message that mentions him.

How to Use

Talking to Felix in Slack

📣 Mention in a channel

Type @Felix followed by your request in any channel Felix is a member of.

@Felix generate 5 new leads for FitnesShock bars in Bangkok
@Felix write a follow-up email in Thai for the gym we contacted last week
@Felix create a sales proposal for a pharmacy chain — 200 bars/month
⚡ Slash command from anywhere

Use /felix from any channel — no need to invite Felix first.

/felix show full price list
/felix how do I hit ฿250,000 this month?
/felix draft outreach email for convenience store chain
💬 Direct Message

DM Felix directly for private conversations. He remembers the thread context.

qualify this lead: hotel chain, 10 locations, wants water for rooms
what's our best margin product right now?
🧵 Thread replies

Felix maintains conversation context within a thread — perfect for back-and-forth on a deal.

@Felix [in a thread] now make it shorter and add Thai version
